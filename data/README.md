# lsc-realtime-interpreter# Data folder rules

- `data/raw/`: videos o capturas originales. NO se sube a GitHub.
- `data/interim/`: keypoints intermedios, sin limpiar. NO se sube a GitHub.
- `data/processed/`: dataset final listo para entrenar (idealmente comprimido y pequeño).
- `data/splits/`: archivos de partición (train/val/test) en CSV.

Privacidad:
- No subir videos de personas a repositorios públicos sin consentimiento.
